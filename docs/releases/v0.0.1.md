# Hum v0.0.1 Release Notes

Date: 2026-07-07

Status: pre-alpha.

Tag: `v0.0.1`

Commit hash: use the verified commit printed by
`tools/check_tag_readiness.ps1` immediately before creating the annotated tag.

## Summary

Hum `0.0.1` is a local pre-alpha foundation snapshot. It establishes the Rust
bootstrap CLI, the Milestone 0 semantic graph surface, core language design
ground truth, editor/schema preview contracts, hygiene checks, and release
discipline.

This is not a public API stability promise.

## Highlights

- Rust bootstrap CLI with no third-party crates and `#![forbid(unsafe_code)]`.
- Checked reference `.hum` surface under `examples/reference_surface.hum`.
- Stable diagnostic codes and JSON surfaces for `hum check`, `hum diagnostics`,
  and `hum explain`.
- `hum.semantic_graph.v0`, `hum.syntax_surface.v0`, `hum.capabilities.v0`,
  `hum.lsp_capabilities.v0`, and `hum.doctor.v0` preview schemas.
- Generated TextMate grammar and editor recovery fixtures.
- Portable setup and public-readiness checks for editor state, local paths,
  UTF-8, no BOM, mojibake, and broken local links.
- Clean-checkout and tag-readiness scripts that verify the committed tree
  before any annotated tag.

## Compatibility Notes

- Version `0.0.1` is pre-alpha; schemas and source syntax are not stable.
- Cargo is the bootstrap build path, not Hum's final distribution identity.
- `hum lsp --capabilities` is a preview discovery command; `hum lsp` server
  mode is not implemented.
- No Hum programs execute in this snapshot.

## Known Risks

- No native backend, interpreter, formatter, package manager, running LSP,
  debugger, or standard library implementation exists yet.
- The language reference and architecture are intentionally ahead of the
  current compiler implementation.
- Release notes require the final verified commit hash from
  `tools/check_tag_readiness.ps1` at tag time.

## Verification Commands

Run before creating the annotated tag:

```powershell
.\tools\check_all.ps1
.\tools\check_clean_checkout.ps1
.\tools\check_tag_readiness.ps1
```

The tag-readiness gate does not create a tag and does not touch remotes. It
prints the exact human tag command only after the checks pass.
